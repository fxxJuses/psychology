# RAG 系统问题记录

## Issue #1: 无法进行全局性文档检索与总结

**日期**: 2026-05-22  
**严重程度**: 中 (架构层面限制)  
**状态**: 待解决

### 问题描述

当前的 RAG 系统基于 chunk 级别检索：将文档切分为固定大小的 chunks（如 500 字符），用户的查询通过向量相似度 / BM25 匹配到最相关的 N 个 chunks，再由 LLM 基于这些 chunks 生成回答。

这种设计在处理**跨段落、全局性**问题时存在根本性局限——chunks 之间相互独立，无法获得文档的全貌。

### 复现用例

```bash
python main.py query "解读一下乌合之众这本书在讲什么" -v
```

### 实际输出

```
根据提供的上下文，文档中并未系统性地解读《乌合之众》全书的具体内容，但间接提到了该书的核心关注点。例如，文档2指出勒庞"有效地阐明了'个人在群体影响下，思想和感觉中道德约束与文明方式突然消失，原始冲动、幼稚行为和犯罪倾向的突然爆发'的实相"。这反映了该书对群体心理中个人行为变化的讨论。此外，文档1和4强调了该书对集体行为和社会心理学思考的巨大影响，以及它"从头到尾表现出一种对重要问题的敏感性"。然而，要全面解读该书内容，需要超出文档范围。因此，文档中未找到完整的解读信息。
```

### 问题分析

1. **Chunk 检索的固有局限**: 检索阶段（retrieval）从 1208 个 chunks 中只召回 top-K（默认 8）个最相关的片段，系统无法感知文档的整体结构和全部内容。

2. **上下文窗口浪费**: 召回的 8 个 chunks 总长度约 2276 字符，但其中包含大量相邻页面内容（如 `第 25 页提要...`、`第 26 页...`），缺少对全书核心观点的概括性描述。

3. **没有查询改写或分解**: 对于"解读一本书"这类高层级问题，系统没有将其分解为"该书的核心论点是什么"、"有哪些主要章节"、"结论是什么"等子问题。

4. **缺少文档级索引**: 当前只有 chunk 级的向量/BM25 索引，没有文档级的摘要索引来支持"这本书讲了什么"这类宏观问题。

### 可能的改进方向

| 方案 | 说明 | 复杂度 |
|------|------|--------|
| **文档摘要索引** | 对每个 PDF 先用 LLM 生成摘要，建立"文档 → 摘要"映射。全局性问题先查摘要，再按需深入 chunks | 中 |
| **层级索引 (Hierarchical Indexing)** | 建立 文档级 → 章节级 → 段落级 的多层索引，逐层缩小检索范围 | 高 |
| **查询分解 (Query Decomposition)** | 将复杂问题拆成子问题分别检索，汇总后回答 | 中 |
| **图谱索引 (Graph RAG)** | 将文档内容构建为知识图谱，支持跨 chunk 的关联推理 | 高 |
| **增大 top-K + 摘要 LLM** | 召回更多 chunks（如 top-50），用 LLM 做全局摘要后再回答问题 | 低 |
| **Map-Reduce 策略** | 对每个 chunk 分别做小回答，再汇总生成最终答案 | 中 |
