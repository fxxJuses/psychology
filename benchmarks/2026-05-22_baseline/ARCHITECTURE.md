### 2.4 检索管线

```
用户问题
    │
    ├── 查询改写（LLM）──→ 生成多个等价查询
    │
    ├── 向量检索（Chroma, top_k=8）  ──┐
    ├── BM25 检索（jieba 分词, top_k=8） ──┤
    │                                      ├── RRF 融合
    │                                      │
    ├── 去重（按 chunk_id）                │
    │
    ├── LLM Rerank（0-10 评分, top_k=4）  ←── DeepSeek-V4-Flash
    │
    └── 格式化上下文 → 送入生成器
```

| 候选检索数 (top_k) | 8 |
| LLM Rerank | 启用，精排后保留 4 条 |

## 3. 评估方法

### 3.1 测试用例

| 类别 | 数量 | 说明 |
|------|------|------|
| factual_lookup | 3 | 事实检索 |

| multi_hop | 2 | 多跳推理 |
| summary | 2 | 总结归纳 |
| negative | 3 | 负样本（语料库外） |
| **合计** | **15** | |

### 3.2 指标定义

| 指标 | 含义 |
|------|------|
| Recall@K | top-K 结果中命中了多少相关文档（查全率） |
| Precision@K | top-K 结果中相关文档的占比（精确率） |
| MRR | 第一个相关文档的排名倒数 |
| NDCG@K | 考虑位置权重的归一化折损累计增益 |
| Hit@K | top-K 中是否至少有一个相关文档 |
| MAP | 平均精确率（Mean Average Precision） |
| Faithfulness | 生成答案是否忠实于上下文 |
| Answer Relevance | 生成答案是否切题 |

### 3.3 Ground Truth 构建

- **方法**：LLM-as-Judge，对每个问题用相同 LLM 逐条判断 top-20 chunk 相关性
- **标注范围**：每种模式取 top-20，合并去重后 ~40 个 chunk/问题
- **偏差风险**：自评偏差（生成和评判使用同一 LLM）

## 4. 已知局限

1. **chunk 级检索无法回答全局问题**（见 `docs/RAG_issue.md`）：如"总结这本书"类问题只有片段视角
2. **Hit@5 不足 60%**：超过 40% 的问题在前 5 个结果中找不到相关信息
3. **Answer Relevance 仅 0.53**：检索召回不足导致生成答案无法完整回答问题
4. **无文档级索引**：三本书之间没有层次化的结构信息
5. **LLM 自评偏差**：后续应考虑用不同模型做裁判或人工抽查
