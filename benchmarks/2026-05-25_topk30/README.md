## 关键发现

1. **排序质量提升（MRR +8~11%）**：更大的候选池 + Reranker 确实把相关文档排得更靠前
2. **命中率提升（Hit@5 +10%、Hit@8 +9%）**：更多用例能在 top-K 中找到至少一条相关文档
3. **Recall 基本持平**：相关 chunk 数量没有增加——扩大候选池引入了更多噪音而非更多相关文档
4. **Recall@1 下降**：更多候选 = 更多噪音稀释了 top-1 信号
5. **BM25 继续退化**：Recall@5 从基线 0.487 → 层级 0.399 → top_k30 0.358，持续下降

### 核心结论

**单纯扩大检索网无法提升 Recall**。问题的根因是 chunk 的语义向量不够精确——同一个主题的不同子话题（如 GAD 的"诊断标准"vs"治疗方法"vs"案例故事"）在 embedding 空间中难以区分。需要 **Contextual Chunking** 来给每个 chunk 注入文档/章节级别的上下文信息。
