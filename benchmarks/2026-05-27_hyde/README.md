## 关键发现

1. **MRR 首次突破 0.6**：HyDE 是所有方案中排序质量最高的。假设文档让 embedding 模型找到了与查询语义更接近的 chunk 邻居
2. **Answer Relevance 首次突破 0.6**：假设文档生成了更丰富的上下文，即使 chunk 不完全匹配，生成质量也有所提升
3. **Recall@5 下降**：当前策略 `queries = [原始查询, 假设文档]` 双查询检索。假设文档偶尔"跑偏"会稀释原始查询的精确匹配
4. **Recall@1 提升 20%**：HyDE 帮最好的结果排到第一位的能力显著增强
